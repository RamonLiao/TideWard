# Project Plan: TideWard (Branding & Documentation Phase)

## 專案核心定位
- **定位**：為 Sui 生態 Lending / Perp 協議設計的自主 AI 風險守護者（Autonomous AI Risk Guardian）。
- **賽道**：Sui Overflow Hackathon — Agentic Web (Autonomous Risk Guardian Sub-track)。
- **核心功能**：即時預言機數據輸入、AI 風險評估、Move Policy Object 自主鏈上參數調整、DAO 治理覆核。

---

## 品牌與文檔進度

### 1. 品牌決定：TideWard
- **Sui 契合度**：與 Sui（水）生態的 `Tide`（潮汐）呼應，比喻市場日常、持續波動。
- **Agent 屬性**：比起 `SurgeAegis` 的被動防守，`TideWard` 強調「主動、持續性」隨市場週期動態調整參數，更契合 Autonomous Agent 定位。

### 2. README.md 建立
- **路徑**：`Tracks/0-Agentic-Web/01-riskguard/README.md`
- **語系**：英式英文（British English）。
- **特點**：以極簡結構呈現 DeFi 風控痛點、TideWard 架構圖（Mermaid）、Sui Move Policy Object 核心優勢，便於評審快速掃描。
